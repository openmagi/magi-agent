# File Send — Web / App Channels

When the user is chatting through the web or mobile app, you deliver files using the `file-send.sh` wrapper — **never** the Telegram Bot API.

## How to know you're in a web/app channel

Look at the top of the conversation for a system message shaped like:

```
[Channel: <name>]
```

Examples of `<name>`: `general`, `news`, `finance`, `daily-update`. If this hint exists, you are in a web/app channel. Use this skill.

If there is **no** `[Channel: ...]` hint, you may be in a Telegram-only bot — use the `telegram-file-output` skill instead.

## The one tool: `file-send.sh`

`file-send.sh` is always in PATH. It:
1. Uploads the local file to the chat-proxy attachment store.
2. Prints an attachment marker like `[attachment:<id>:<filename>]`.
3. You include that marker verbatim in your chat reply so the client renders it.

### Signature
```
file-send.sh <absolute-path-to-file> [channel-name]
```
- `channel-name` defaults to `general`.
- Always pass the exact name from the `[Channel: ...]` hint.

## Examples (copy-paste)

### Example 1 — Markdown report

```bash
# [Channel: general] in system context
cat > /tmp/report.md << 'FILEEOF'
# Weekly Summary
- Revenue up 12%
- New users: 1,204
FILEEOF

MARKER=$(file-send.sh /tmp/report.md general)
echo "$MARKER"      # [attachment:7f3c...:report.md]
rm /tmp/report.md
```

Then in your chat reply:
```
주간 요약 보고서입니다.
[attachment:7f3c...:report.md]
```

### Example 2 — CSV export to the active channel

```bash
# [Channel: finance]
cat > /tmp/trades.csv << 'FILEEOF'
date,symbol,qty,price
2026-04-17,AAPL,10,195.23
2026-04-18,NVDA,3,920.11
FILEEOF

MARKER=$(file-send.sh /tmp/trades.csv finance)
echo "$MARKER"
rm /tmp/trades.csv
```

### Example 3 — Image (inline preview on client)

```bash
# [Channel: news]
MARKER=$(file-send.sh /tmp/chart.png news)
echo "$MARKER"
```

Client renders jpg/png/gif/webp inline; other types render as a file card.

### Example 4 — XLSX generated by a script

```bash
# [Channel: general]
python3 /home/ocuser/scripts/build_workbook.py --out /tmp/summary.xlsx
MARKER=$(file-send.sh /tmp/summary.xlsx general)
echo "$MARKER"
rm /tmp/summary.xlsx
```

### Example 5 — PDF from a report pipeline

```bash
# [Channel: legal]
MARKER=$(file-send.sh /tmp/contract.pdf legal)
echo "$MARKER"
```

## When to send as file

| Situation | Action |
|-----------|--------|
| User asks for raw file (.md, .csv, .json, .pdf, .xlsx, etc.) | **Always send as file** |
| Output exceeds ~3000 characters | **Send as file** |
| Research reports, analysis documents | **Send as file** |
| Code files, scripts, configs | **Send as file** |
| Structured data (tables, lists, datasets) | **Send as file** |
| User says "send file", "attach", "download", "파일로 줘", "엑셀로", "PDF로" | **Send as file** |
| Short conversational answer | Send as text (normal) |

## Hard rules

1. **If `[Channel: <name>]` is present, you MUST use `file-send.sh <path> <name>`.** Calling `api.telegram.org/.../sendDocument` in a web/app channel delivers the file to the wrong place and the user never sees it.
2. **Never use the `message` tool's `send` action for files in web/app channels** — it always fails.
3. Always include the `[attachment:...]` marker in your reply text so the client renders it.
4. Clean up `/tmp` files after sending.
5. Max file size: 50MB.
6. Do not finish a turn that created a user-facing file by only saying the workspace path or a `fileRead:` reference. The attachment marker is the delivery handle the client can render.

## Supported file types

- **Images (inline):** jpg, png, gif, webp
- **Documents (file card):** pdf, txt, csv, md, html, json, docx, xlsx, zip

## Fallback

If `file-send.sh` is unavailable (e.g. `command not found`), fall back to inline text output and tell the user the attachment pipeline is temporarily down. Do **not** switch to Telegram API as a fallback when in a web/app channel — it sends to the wrong destination.

# What Works Today

A scannable map of what the local `magi` CLI / `magi-agent serve` can actually do right now, what ships in shadow/observe-only mode by design, and what is still planned.

Two things to hold in mind while reading:

1. **"Default-off" describes the enforcement/governance layer**, the boundary modules that can block, modify, or gate agent behavior. It is **not** a statement about whether the agent can do work. With a provider key configured, the local runtime executes real model calls and a deep stack of first-party tools today. The shadow posture exists because untested enforcement is worse than no enforcement; see [Boundaries](/docs/boundaries). A feature marked 🚧 below is still fully functional: it is its enforcement/governance boundary that runs observe-only, so the agent does the work while the governance layer records without blocking.
2. **A clean local install applies the local full-runtime overlay by default.** Both `magi` (CLI/TUI) and `magi-agent serve` apply the `LOCAL_FULL_RUNTIME_ENV_DEFAULTS` overlay at startup, which seeds dozens of capability flags ON (subagents, work queue, learning, memory capture, self-review, compaction, vault, user hooks, keyless web fetch, and more). [^0] This overlay is distinct from the named `full` runtime *profile*: a Homebrew install seeds `MAGI_RUNTIME_PROFILE=full`, but the overlay is applied for any profile *except* the conservative ones (`safe`, `minimal`, `off`, `conservative`, `eval`), and can be turned off with `MAGI_AGENT_LOCAL_FULL_RUNTIME_DEFAULTS=0`. It is **self-host / local only**: hosted multi-tenant deployments do **not** apply it and stay conservative. So the "Works today" column below is what a default local install gives you, not the library-level code default (which stays conservative for import-time tests and custom deployments).

### ✅ Works today

**Model + tools**

- **Real model calls** across five providers (Anthropic, OpenAI, Gemini/Google, Fireworks, OpenRouter) through LiteLlm, once a provider key (`ANTHROPIC_API_KEY` / `OPENAI_API_KEY` / `GEMINI_API_KEY` | `GOOGLE_API_KEY` / `FIREWORKS_API_KEY` / `OPENROUTER_API_KEY`) or `~/.magi/config.toml` is present. With no key, the CLI still launches against a model-free stub. [^1]
- **First-party local tools** (file read/write/edit, patch apply, Bash) are exposed by default on the real-runner path. [^2]
- **Coding-quality stack** runs on by default in the local profile: format-on-write, LSP diagnostics after edits, ripgrep search, a read ledger that scores edit confidence, and edit-retry reflection that feeds a failed edit back into the next turn. (Fuzzy edit matching ships too, but is on in the `eval` profile rather than the default local overlay.) [^8]
- **Resilience controls** are on: loop guard, max-steps brake, error recovery with retry, provider repair, and a coding repair loop that re-invokes the model when verification fails. [^8]
- **Tool concurrency** (up to 8 parallel tool calls) and the deferred-tool seam are on. [^8]
- **Persistent Python + code-action** are default-ON in the full profile: a stateful Python interpreter tool (the namespace persists across calls in a turn) plus a code-action surface, alongside the file tools. [^41]
- **Resilience** additions are on: one-shot empty-response recovery and a bounded escalation, proactive compaction near context overflow, reconnect/circuit-breaker resilience for MCP endpoints and Telegram polling, and tool-exception reflection (a raising tool becomes a model-visible corrective `tool_result` with retry guidance instead of killing the turn; `FileEdit`/`PatchApply` keep their specialized edit-retry handler). [^43]
- **Anthropic prompt cache** is default-ON on the live ADK Anthropic path, so cache breakpoints are added automatically when an Anthropic model is used. [^19]
- **`--resume` / `--continue`** rehydrate a prior session from the on-disk JSONL transcript by default in the local-full profile (`MAGI_CLI_RESUME_ENABLED=1`); the eval profile and an explicit opt-out stay id-only. [^10]
- **Permission gating** in Claude-Code style: `default`, `acceptEdits`, and `bypassPermissions` modes decide when a tool may run (see the [CLI reference](/docs/cli)). [^2] When `--permission-mode` is omitted, local CLI runs default to `bypassPermissions`; pass `--permission-mode default` to opt into per-tool approval prompts. [^2]

**Agent loop**

- **Subagents.** The live child runner is enabled by default locally, so a `SpawnAgent` call boots a real model-backed child and returns its answer to the parent. The child's toolset defaults to **read-only** (`MAGI_CHILD_RUNNER_TOOLSET=readonly`), so children research and reason but do not write files unless you widen the toolset. [^4]
- **Background tasks / work queue.** With the workflow executor on, you can enqueue background work that a child runner executes in the shared workspace and whose result is injected back into the conversation. The web `/workflows` view and the TUI `/tasks` surface the queue. [^5]
- **Sessions** plus both the **headless** NDJSON surface and the **interactive TUI** (with `/` [slash commands](/docs/cli-commands)), and the **local HTTP dashboard** via `magi-agent serve` (Customize, Integrations, Credentials, Learning, and Work-Queue tabs).

**Memory, learning, review**

- **Memory.** The memory master (`MAGI_MEMORY_ENABLED`) is now on in the local overlay, so the full loop runs by default: a concise entry is written to `memory/daily/YYYY-MM-DD.md` at turn-finalization, the compaction tree rollups build, and prior memories are recalled (loop-aware re-rank on) at prompt-build time. Capture is fail-soft and skips trivial/errored/incognito turns. Only session-end extraction stays off by default. [^6]
- **Learning + skill curation.** Reflection runs live and extracts learning candidates from the transcript into the local learning store; the skill curator's archive janitor retires stale agent-proposed items (never human-approved ones). Both are live, not shadow. [^15]
- **Self-review.** A non-blocking review fork runs after a turn with a restricted toolset and emits review candidates; it is live (`MAGI_SELF_REVIEW_SHADOW=0`), not shadow. [^17]
- **Context compaction.** Real tail-keep compaction is wired via an ADK `before_model_callback`: it estimates per-message token cost and trims to a tail once the budget is exceeded. Summarize/prune strategies remain lab-experimental. [^18]

**Verification + extension**

- **Pre-final completion/evidence gate** is default-ON and, on coding turns, blocks output with a `pre_final_evidence_gate_blocked` terminal error when required evidence is missing and the repair loop cannot satisfy it. Read-only turns skip the gate (byte-identical path). [^20]
- **Evidence ledger.** Structured tool/skill/spawn recording is default-ON; it records (producer), and enforcement happens downstream at the pre-final gate, not at producer time. The live coding handler now emits **typed** records (EditMatch, CodeDiagnostics, GitDiff, TestRun, Calculation, CommitCheckpoint) extracted from tool results, so an emitted `public_ref` can satisfy a required evidence ref at the gate. [^21]
- **No-fork pack extension.** Disk packs in `~/.magi/packs/` and `<cwd>/.magi/packs/` (plus the bundled first-party packs) are discovered and loaded through the same path as first-party. A user pack can add a new primitive, override a first-party ref, or disable a pack. A **validator** authored in any loaded pack is merged into the live pre-final enforcement gate. Discovery/load is unconditional (no master flag) and fail-open when the packs tree is empty. [^22]
- **Customize verification + custom rules.** The dashboard Customize tab's verification presets and custom rules are profile-aware default-ON in the full local profile (OFF under `safe`/`eval`). Persisted overrides in `~/.magi/customize.json` translate into the recipe-driven pre-final evidence gate; with no authored overrides this is byte-identical (a no-op until you configure it). The newer lifecycle hooks, mutators, and shell-action kinds are separate lab-experimental opt-ins (see Default-off). [^29]
- **Policies + evidence-grounded judges.** A Policy is a composition of custom rules with full CRUD (`/v1/app/policies`), authored by hand, by one-shot natural language, or by a multi-turn conversational compiler (all default-ON in the full profile, fail-open with no provider key). An `llm_criterion` rule's judge now receives the turn's evidence ledger as grounding; a deterministic evidence producer plus a fail-closed session-evidence gate at `before_tool_use` can require a named evidence record before a tool runs; and a **mode** can scope which policies/rules (and a tighten-only `permission_mode`) apply for a turn. Enforcement is default-ON but inert until you author a policy and scope it to an active mode. [^42]

**Web + browser + credentials + hooks**

- **WebFetch** works keyless out of the box: the local overlay enables live web acquisition with jina-reader (keyless; an optional key only raises rate limits) and insane-fetch (a local `curl_cffi` path), so a fresh user gets a working URL-to-content path with zero keys. [^13]
- **Browser tool** is enabled by default in the local profile; it needs the optional `magi-agent[browser]` extra plus `playwright install chromium` to actually drive a browser. [^23]
- **Local credential vault + injecting proxy** run for the local serve overlay so the dashboard "Credentials" registration works out of the box and the bot can *use* a registered credential without seeing it. The proxy needs mitmproxy (bundled via the `vault` extra); if it is missing, serve logs a hint and continues without the proxy. Self-host only: hosted bots never run this overlay. [^24]
- **Claude-Code-style user hooks** read `~/.magi/settings.json` and `<workspace>/.magi/settings.json` and run user-supplied bash/http/llm at lifecycle points. On for self-host, a no-op until you author hooks. Self-host only, never applied in hosted multi-tenant. The related **external hooks** surface (LLM-classified + HTTP webhooks at lifecycle points) is also profile default-ON. [^25]
- **Third-party pack activation.** User-authored pack primitives now activate at runtime by default in the full profile: tool packs (with a runnable inline tool-handler ABI, so `magi pack new tool` yields a live tool), validator packs, evidence-producer packs, and recipe-as-code via `spec_callable`. (Pack capability-token enforcement stays default-OFF, see Default-off.) [^31]
- **Apify Actors.** Actor *discovery* (`apify_search_actors`) is keyless and registered in the default native plugin set; *running* an Actor (`apify_run_actor`) needs your own `APIFY_TOKEN`. [^27]
- **Autopilot** (an Interview/Plan/Execute/Review FSM) and the **observability event store** are on by default in the local profile. [^26]

**Knowledge, sessions, durability, audit**

- **Local knowledge base.** A native `KnowledgeSearch` tool is registered by default and searches the workspace KB (`<workspace>/knowledge/` and `.magi/knowledge/`). It ranks with qmd when available and otherwise falls back to a dependency-free keyword scan; `magi knowledge init` builds an optional qmd index (add `--vector` for embeddings). [^35]
- **Session transcript log.** Each turn is now written to `~/.magi/projects/<cwd-slug>/<session_id>.jsonl` (one sanitized event per line, files `0600`). The transcript write path is on by default, and `--resume` / `--continue` rehydration off it is also on by default in the local-full profile. [^36]
- **Durable run substrate.** A local sqlite checkpoint store writes by default, checkpoints emit from the headless tap, and a boot recovery sweep runs at startup: it reclaims interrupted background tasks at-least-once. Foreground auto-continuation of an interrupted main turn is deliberately off (opt-in), so a clean boot is unchanged. [^37]
- **Chat Audit panel.** A default-ON read surface in the dashboard shows per-session policy-enforcement verdicts (grouped by run), projected read-only over the observability store; it produces no new verdicts. A Workspace tab likewise lists workspace files. [^38]

### 🚧 Default-off / shadow today

Self-host users who want to exercise everything below at once can `source scripts/dogfood-full-on.env`, the opt-in dogfood profile that turns on every public default-OFF capability flag (a reverse completeness test keeps it in sync with the flag registry). It is never the default, and it does not relax the profile-exempt hard-safety items (e.g. computer-use). [^34]

- **Evidence / governance enforcement boundaries** (the rollout boundary modules) run in observe-only / local-fake mode; the ledger records, but those boundary verdicts are not attached to live traffic. `EvidenceRolloutMetadata.traffic_attached` / `execution_attached` are `Literal[False]`. The one boundary that already blocks today is the pre-final evidence gate (listed under Works today). [^3]
- **Research / citation governance.** The local overlay sets `MAGI_RESEARCH_GOVERNANCE_MODE=audit`: a deterministic citation audit runs and persists false-positive data but never mutates or blocks. `enforce` mode (which would re-prompt a bounded number of times) is opt-in (the dogfood/lab profile turns it on). [^9]
- **Memory session-end extraction.** Daily capture, compaction rollups, and recall are now on by default (see Works today); only the session-end extraction pass (`MAGI_MEMORY_SESSION_EXTRACT_ENABLED`) still follows a separate default-OFF flag. [^6]
- **Durable foreground continuation.** The durable substrate reclaims interrupted *background tasks* by default, but auto-continuing an interrupted *foreground* turn (`MAGI_DURABLE_FOREGROUND_CONTINUATION_ENABLED`) is opt-in; when off, the boot sweep does not resume the main turn. [^37]
- **Scheduled background execution.** The gateway work-queue dispatcher tick is on (`MAGI_WORK_QUEUE_EXECUTOR_ENABLED=1`), but its inner runner is a safe stub unless `MAGI_BACKGROUND_LIVE_RUNNER_ENABLED=1`, so queued jobs handled by the daemon do not run for real by default. (In-turn background tasks via the workflow executor + live child runner do run, see Works today.) [^5]
- **First-run onboarding wizard.** `MAGI_ONBOARDING_WIZARD_ENABLED` is default-OFF; when on, the dashboard guides provider/key/model setup and writes `~/.magi/config.toml`. [^40]
- **SHACL deterministic verifier.** A pack can compile natural-language or `raw.ttl` rules into a deterministic SHACL verifier that runs at the pre-final gate, but it is a lab-experimental flag, default-OFF. (The plain `llm_criterion` custom-rule judge is default-ON, see Policies under Works today.) [^11]
- **Recipe engine & kernel packs.** Recipe selection via the in-context `select_recipe` tool is default-OFF (the registration is a no-op unless `MAGI_RECIPE_ROUTING_LLM_ENABLED` is set). No-fork kernel recipe packs (`MAGI_KERNEL_RECIPE_PACKS_ENABLED`) and role provides (`MAGI_KERNEL_ROLE_PROVIDES_ENABLED`) are merged but gated as lab-experimental; an external role is a scope label only. A new verifier stage or lifecycle hook point remains an upstream (immutable-core) change. [^7]
- **Always-on gateway daemon.** `magi gateway start --once` runs a single scheduler tick today; the supervising always-on daemon is gated behind `MAGI_GATEWAY_DAEMON_ENABLED` (default OFF), and each watcher still respects its own gate. [^12]
- **WebSearch engine results.** URL fetch is keyless (above), but query-to-results `WebSearch` needs a search provider key (Brave / SerpAPI) or falls back to driving the browser tool. [^13]
- **Computer-use / desktop GUI control** (macOS): strictly opt-in via `MAGI_COMPUTER_TOOL_ENABLED`. Unlike the browser tool this is **profile-exempt**: it is never enabled by any profile default (not even `lab`), because it drives the real desktop with no sandbox. Needs the optional desktop extra. [^14]
- **Runner-policy route scoping.** The runner computes per-turn capability/route scoping (`MAGI_RUNNER_POLICY_ROUTING_ENABLED=1`), but route-blocking is off (`MAGI_RUNNER_POLICY_ROUTE_BLOCKING_ENABLED=0`), so the scoping is audit metadata and does not block a turn. [^28]
- **External channel delivery** (Telegram / Discord live send) and **external integrations** (Composio). Composio is off until enabled; `MAGI_COMPOSIO_CREDENTIAL_SOURCE` then selects the credential mode: the **zero-key `platform` broker** (a `MAGI_PLATFORM_API_KEY` brokers tool calls through the openmagi platform with no Composio key, and the dashboard Composio card now defaults to this brokered path), or your own `COMPOSIO_API_KEY` (env mode, demoted to an Advanced fallback in the card). It is also the path that bridges MCP servers into the toolset today. Telegram easy-setup needs `MAGI_TELEGRAM_EASY_SETUP_ENABLED` plus `TELEGRAM_API_ID` / `TELEGRAM_API_HASH`. [^16]
- **Customize lifecycle extensions.** Beyond the default-ON verification/custom-rules above, the wizard exposes audit-only `custom_rule` gate sites at many lifecycle points (user-prompt-submit, subagent-stop, turn boundaries, per-LLM-call, compaction, task-checkpoint, artifact-created, session/task emitters), two mutator kinds (`prompt_injection`, `output_rewrite`), and operator subprocess actions (`shell_command`, `shell_check`). Each is a separate lab-experimental flag, default-OFF, and a no-op until you author rules and opt in; the `on_session_end` slot is honest-degrade in v1 (the wizard shows it, the runtime emit wire ships in a follow-up). [^30]
- **Pack capability-token enforcement.** Pack primitive activation (tool / validator / evidence-producer packs, recipe-as-code) is now default-ON (see Works today), but enforcing declared capability tokens at the ABI surface (`MAGI_PACK_CAPABILITY_ENFORCEMENT_ENABLED`, defense-in-depth, not isolation) is still default-OFF. [^31]

### ❌ Planned

- Live enforcement authority attached to real traffic and decisions ("Stage 3"). See [Default-Off Gates](/docs/default-off-gates) for the Stage 1/2/3 definitions and promotion criteria. [^3]
- Cross-boundary repair orchestration across multiple contracts.

### Documented but not wired (reference contracts)

These ship as fully-built, tested specifications with a machine-readable `REFERENCE_CONTRACT = True` marker (or an equivalent dormant-by-design note), but have **no live OSS runtime caller** today. Read them as the intended contract, not as behavior you get on a turn.

- **Spend-cap / quota / reservation FSM** (`magi_agent/billing/`). A fail-closed billing FSM (reserve / commit / release / evaluate-quota), but `SpendCapProbe` is a Protocol placeholder and nothing in the OSS loop calls it. [^32]
- **Agent job-queue lifecycle FSM** (`magi_agent/ops/job_queue.py`). A durable-looking job lifecycle (queued, running, completed, failed, dead-lettered, cancelled, timed-out) that is in-memory only with zero non-test callers; the genuine durable queue is the work queue. [^32]
- **Ledger orchestrator answer assembly** (`magi_agent/recipes/ledger_orchestrator.py`). Benchmark-only and dormant behind `MAGI_LEDGER_ORCHESTRATOR_ENABLED` (default-OFF), with no default caller. [^33]
- **Memory search backend protocol** (`magi_agent/memory/search/base.py`). The pluggable recall-search Protocol is "not wired into the agent loop"; live recall is itself gated by the memory master (see Default-off). [^33]

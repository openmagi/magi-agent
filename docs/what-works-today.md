# What Works Today

A scannable map of what the local `magi` CLI / `magi-agent serve` can actually do right now, what ships in shadow/observe-only mode by design, and what is still planned.

Two things to hold in mind while reading:

1. **"Default-off" describes the enforcement/governance layer**, the boundary modules that can block, modify, or gate agent behavior. It is **not** a statement about whether the agent can do work. With a provider key configured, the local runtime executes real model calls and a deep stack of first-party tools today. The shadow posture exists because untested enforcement is worse than no enforcement; see [Boundaries](/docs/boundaries). A feature marked 🚧 below is still fully functional: it is its enforcement/governance boundary that runs observe-only, so the agent does the work while the governance layer records without blocking.
2. **A clean local install applies the local full-runtime overlay by default.** Both `magi` (CLI/TUI) and `magi-agent serve` apply the `LOCAL_FULL_RUNTIME_ENV_DEFAULTS` overlay at startup, which seeds dozens of capability flags ON (subagents, work queue, learning, memory capture, self-review, compaction, vault, user hooks, keyless web fetch, and more). [^0] This overlay is distinct from the named `full` runtime *profile*: a Homebrew install seeds `MAGI_RUNTIME_PROFILE=full`, but the overlay is applied for any profile *except* the conservative ones (`safe`, `minimal`, `off`, `conservative`, `eval`), and can be turned off with `MAGI_AGENT_LOCAL_FULL_RUNTIME_DEFAULTS=0`. It is **self-host / local only**: hosted multi-tenant deployments do **not** apply it and stay conservative. So the "Works today" column below is what a default local install gives you, not the library-level code default (which stays conservative for import-time tests and custom deployments).

### ✅ Works today

**Model + tools**

- **Real model calls** across five providers (Anthropic, OpenAI, Gemini/Google, Fireworks, OpenRouter) through LiteLlm, once a provider key (`ANTHROPIC_API_KEY` / `OPENAI_API_KEY` / `GEMINI_API_KEY` | `GOOGLE_API_KEY` / `FIREWORKS_API_KEY` / `OPENROUTER_API_KEY`) or `~/.magi/config.toml` is present. With no key, the CLI still launches against a model-free stub. [^1]
- **First-party local tools** (file read/write/edit, patch apply, Bash) are exposed by default on the real-runner path. [^2]
- **Coding-quality stack** runs on by default in the local profile: format-on-write, LSP diagnostics after edits, ripgrep search, a read ledger that scores edit confidence, and edit-retry reflection that feeds a failed edit back into the next turn. (Fuzzy edit matching ships too, but is on in the `eval` profile rather than the default local overlay.) [^8]
- **Resilience controls** are on: loop guard, max-steps brake, error recovery with retry, provider repair, and a coding repair loop that re-invokes the model when verification fails. [^8]
- **Tool concurrency** (up to 8 parallel tool calls) and the deferred-tool seam are on. [^8]
- **Permission gating** in Claude-Code style: `default`, `acceptEdits`, and `bypassPermissions` modes decide when a tool may run (see the [CLI reference](/docs/cli)). [^2] When `--permission-mode` is omitted, local CLI runs default to `bypassPermissions`; pass `--permission-mode default` to opt into per-tool approval prompts. [^2]

**Agent loop**

- **Subagents.** The live child runner is enabled by default locally, so a `SpawnAgent` call boots a real model-backed child and returns its answer to the parent. The child's toolset defaults to **read-only** (`MAGI_CHILD_RUNNER_TOOLSET=readonly`), so children research and reason but do not write files unless you widen the toolset. [^4]
- **Background tasks / work queue.** With the workflow executor on, you can enqueue background work that a child runner executes in the shared workspace and whose result is injected back into the conversation. The web `/workflows` view and the TUI `/tasks` surface the queue. [^5]
- **Sessions** plus both the **headless** NDJSON surface and the **interactive TUI** (with `/` [slash commands](/docs/cli-commands)), and the **local HTTP dashboard** via `magi-agent serve` (Customize, Integrations, Credentials, Learning, and Work-Queue tabs).

**Memory, learning, review**

- **Memory capture.** On the live local-chat turn-finalization path, a concise entry is written to `memory/daily/YYYY-MM-DD.md` by default, because the local overlay sets `MAGI_MEMORY_WRITE_ENABLED=1` and the turn hook gates on that write flag. Capture is fail-soft and skips trivial/errored/incognito turns. The compaction tree rollups, recall, and session-end extraction are **not** on by default (see Default-off). [^6]
- **Learning + skill curation.** Reflection runs live and extracts learning candidates from the transcript into the local learning store; the skill curator's archive janitor retires stale agent-proposed items (never human-approved ones). Both are live, not shadow. [^15]
- **Self-review.** A non-blocking review fork runs after a turn with a restricted toolset and emits review candidates; it is live (`MAGI_SELF_REVIEW_SHADOW=0`), not shadow. [^17]
- **Context compaction.** Real tail-keep compaction is wired via an ADK `before_model_callback`: it estimates per-message token cost and trims to a tail once the budget is exceeded. Summarize/prune strategies remain lab-experimental. [^18]

**Verification + extension**

- **Pre-final completion/evidence gate** is default-ON and, on coding turns, blocks output with a `pre_final_evidence_gate_blocked` terminal error when required evidence is missing and the repair loop cannot satisfy it. Read-only turns skip the gate (byte-identical path). [^20]
- **Evidence ledger.** Structured tool/skill/spawn recording is default-ON; it records (producer), and enforcement happens downstream at the pre-final gate, not at producer time. [^21]
- **No-fork pack extension.** Disk packs in `~/.magi/packs/` and `<cwd>/.magi/packs/` (plus the bundled first-party packs) are discovered and loaded through the same path as first-party. A user pack can add a new primitive, override a first-party ref, or disable a pack. A **validator** authored in any loaded pack is merged into the live pre-final enforcement gate. Discovery/load is unconditional (no master flag) and fail-open when the packs tree is empty. [^22]

**Web + browser + credentials + hooks**

- **WebFetch** works keyless out of the box: the local overlay enables live web acquisition with jina-reader (keyless; an optional key only raises rate limits) and insane-fetch (a local `curl_cffi` path), so a fresh user gets a working URL-to-content path with zero keys. [^13]
- **Browser tool** is enabled by default in the local profile; it needs the optional `magi-agent[browser]` extra plus `playwright install chromium` to actually drive a browser. [^23]
- **Local credential vault + injecting proxy** run for the local serve overlay so the dashboard "Credentials" registration works out of the box and the bot can *use* a registered credential without seeing it. The proxy needs mitmproxy (bundled via the `vault` extra); if it is missing, serve logs a hint and continues without the proxy. Self-host only: hosted bots never run this overlay. [^24]
- **Claude-Code-style user hooks** read `~/.magi/settings.json` and `<workspace>/.magi/settings.json` and run user-supplied bash/http/llm at lifecycle points. On for self-host, a no-op until you author hooks. Self-host only, never applied in hosted multi-tenant. [^25]
- **Apify Actors.** Actor *discovery* (`apify_search_actors`) is keyless and registered in the default native plugin set; *running* an Actor (`apify_run_actor`) needs your own `APIFY_TOKEN`. [^27]
- **Autopilot** (an Interview/Plan/Execute/Review FSM) and the **observability event store** are on by default in the local profile. [^26]

### 🚧 Default-off / shadow today

- **Evidence / governance enforcement boundaries** (the rollout boundary modules) run in observe-only / local-fake mode; the ledger records, but those boundary verdicts are not attached to live traffic. `EvidenceRolloutMetadata.traffic_attached` / `execution_attached` are `Literal[False]`. The one boundary that already blocks today is the pre-final evidence gate (listed under Works today). [^3]
- **Research / citation governance.** The local overlay sets `MAGI_RESEARCH_GOVERNANCE_MODE=audit`: a deterministic citation audit runs and persists false-positive data but never mutates or blocks. `enforce` mode (which would re-prompt a bounded number of times) is opt-in (the dogfood/lab profile turns it on). [^9]
- **Memory rollups + recall + session extraction.** Daily capture is on, but the compaction tree build (weekly/monthly/ROOT), memory recall at prompt-build time, and session-end extraction follow the (default-OFF) memory master and are not wired in the local overlay. Set `MAGI_MEMORY_ENABLED=1` (or `[memory] enabled=true` in `~/.magi/config.toml`) to light them up. [^6]
- **CLI session transcript log + resume.** `MAGI_CLI_SESSION_LOG_ENABLED` and `MAGI_CLI_RESUME_ENABLED` are explicitly staged OFF ("0") in the local overlay, so `--resume` / `--continue` rehydration is not active yet; a later release stages the write path and rehydration on together. [^10]
- **SHACL deterministic verifier & custom-rule LLM criterion gate.** A pack can compile natural-language or `raw.ttl` rules into a deterministic SHACL verifier, and an LLM-criterion rule can block at the pre-final gate, but both are lab-experimental flags and default-OFF. [^11]
- **Recipe engine & kernel packs.** Recipe selection via the in-context `select_recipe` tool is default-OFF (the registration is a no-op unless `MAGI_RECIPE_ROUTING_LLM_ENABLED` is set). No-fork kernel recipe packs (`MAGI_KERNEL_RECIPE_PACKS_ENABLED`) and role provides (`MAGI_KERNEL_ROLE_PROVIDES_ENABLED`) are merged but gated as lab-experimental; an external role is a scope label only. A new verifier stage or lifecycle hook point remains an upstream (immutable-core) change. [^7]
- **Always-on gateway daemon.** `magi gateway start --once` runs a single scheduler tick today; the supervising always-on daemon is gated behind `MAGI_GATEWAY_DAEMON_ENABLED` (default OFF), and each watcher still respects its own gate. [^12]
- **WebSearch engine results.** URL fetch is keyless (above), but query-to-results `WebSearch` needs a search provider key (Brave / SerpAPI) or falls back to driving the browser tool. [^13]
- **Computer-use / desktop GUI control** (macOS): strictly opt-in via `MAGI_COMPUTER_TOOL_ENABLED`. Unlike the browser tool this is **profile-exempt**: it is never enabled by any profile default (not even `lab`), because it drives the real desktop with no sandbox. Needs the optional desktop extra. [^14]
- **Runner-policy route scoping.** The runner computes per-turn capability/route scoping (`MAGI_RUNNER_POLICY_ROUTING_ENABLED=1`), but route-blocking is off (`MAGI_RUNNER_POLICY_ROUTE_BLOCKING_ENABLED=0`), so the scoping is audit metadata and does not block a turn. [^28]
- **External channel delivery** (Telegram / Discord live send) and **external integrations** (Composio). Composio needs a key and an explicit enable; it is also the path that bridges MCP servers into the toolset today. Telegram easy-setup needs `MAGI_TELEGRAM_EASY_SETUP_ENABLED` plus `TELEGRAM_API_ID` / `TELEGRAM_API_HASH`. [^16]
- **Anthropic prompt-cache model path.** The message-cache flag is on in the local overlay and the cache-aware model class is wired, but it only takes effect when a Claude/Anthropic model id is actually routed through the ADK model path; it is not a no-op, just not on the default route for every provider. [^19]

### ❌ Planned

- Live enforcement authority attached to real traffic and decisions ("Stage 3"). See [Default-Off Gates](/docs/default-off-gates) for the Stage 1/2/3 definitions and promotion criteria. [^3]
- Cross-boundary repair orchestration across multiple contracts.
- Additional external authority for managed systems.

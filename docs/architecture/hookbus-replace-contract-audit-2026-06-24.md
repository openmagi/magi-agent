# HookBus replace-contract audit (2026-06-24)

Per-event audit of which HookBus events accept a `HookResult(action='replace', value=...)` reply today, the exact emitter site, and what extension each event needs before the F-MUT customize series can wire prompt-injection and output-rewrite primitives safely.

## Purpose

The F-MUT (mutation-kind) wizard outputs declarative authoring kinds that mutate runtime payloads instead of merely gating them. Two of those kinds, `prompt_injection` and `output_rewrite`, depend on the underlying hook emitter projecting a `replace` reply onto a concrete runtime payload (prompt sections, LLM request fields, LLM response content, tool arguments, tool result, recovery directive). This audit enumerates which of the eight live hook events accept that contract today, which silently ignore it, and which fail closed. It is the precondition for the F-MUT series so that wizard outputs do not produce dead authoring kinds that look valid in the UI but never affect runtime.

The matrix below is the source of truth for the wizard's authoring-kind catalog. Each row is one of the eight events that has a real emitter site; events present in the manifest but without an emitter are excluded (those are the F-UX1 lifecycle audit finding from the customize firing tests). The audit was produced by reading every emit site, every consumer, and every `HookBus.run` / `HookBus.run_async` call path; results are not derived from the manifest alone.

## Claude Code parity

Claude Code's `settings.json` hook contract returns `{decision: approve | deny | ask, message, additionalContext, hookSpecificOutput, updatedInput}`. By spec, Claude Code does not project `updatedInput` onto tool arguments (it is advisory) and does not rewrite tool results. The hook contract is therefore gate-plus-append, never gate-plus-replace.

Magi's parse layer at `magi_agent/hooks/settings_loader.py` (`command_executor`) is richer than CC at the parse layer: it maps `updatedInput` onto `HookResult(action='replace', value=<obj>)` and parses `permissionDecision` into `permission_decision` / `decision`. The strictly emitter-driven story is narrower today: only one event consumes the `replace` reply. That event is `BEFORE_SYSTEM_PROMPT`, where `magi_agent/runtime/message_builder.py:771` (`_apply_prompt_transform`) substitutes the prompt-section list from `HookResult.value` and reasserts hard-safety sections afterward. Claude Code has `UserPromptSubmit` with `additionalContext` append, not full-section replace; this is the single live behavioral difference today.

For the other seven events, magi is functionally at CC parity from the model's point of view: block and permission decisions only; `replace` is parsed by the command executor but either ignored (the `facades.py` and `hook_wiring` tool sites) or fails closed (the `callback_adapter.py` LLM, turn, and error sites). There is also a parity gap in the other direction: `HookResult.metadata['additionalContext']` bytes are stored on the result but never delivered to the model, so even the append-only contract that CC honors is not yet wired in magi.

## Extensions needed

The minimum set of signature changes required so the F-MUT wizard can emit declarative `prompt_injection` and `output_rewrite` kinds without producing dead-code authoring outputs:

1. `magi_agent/facades.py` (`execute_tool_with_hooks`): project `before_result.final_action == 'replace'` onto a new arguments dict using the typed payload `BeforeToolUseReplace{arguments}`. Same shape for `AFTER_TOOL_USE` to rewrite `ToolResult` via `AfterToolUseReplace{result_text, structured_data, status}`.
2. `magi_agent/cli/hook_wiring.py` (`_build_before_tool_bridge` / `_build_after_tool_bridge`): stop ignoring `replace`; project onto ADK `tool_context` arguments and `tool_response` respectively so `settings.json` `updatedInput` is honored end-to-end.
3. `magi_agent/adk_bridge/callback_adapter.py` (`run_invocation`): replace the blanket `OpenMagiAdkCallbackBlocked` with a per-point projector dispatch table for `action == 'replace'`: `BEFORE_TURN_START` projects a state/session patch; `BEFORE_LLM_CALL` mutates the LLM request (messages, system, tool_choice, model, temperature); `AFTER_LLM_CALL` rewrites the LLM response; `AFTER_TURN_END` rewrites outbound assistant text and follow-ups; `ON_ERROR` projects a recovery directive.
4. Plumb `HookResult.metadata['additionalContext']` from `BEFORE_TOOL_USE`, `AFTER_TOOL_USE`, and the user-prompt-submit analog (`BEFORE_SYSTEM_PROMPT`) into the next-turn message stream so the CC-style `additionalContext` append actually reaches the model. This is a parity fix, not net new.
5. Define and document typed replace payload schemas per event under `magi_agent/hooks/replace_payloads.py` (pydantic models): `BeforeToolUseReplace{arguments}`, `AfterToolUseReplace{result_text, structured_data, status}`, `BeforeLlmCallReplace{messages, system, tool_choice, temperature, model}`, `AfterLlmCallReplace{content, tool_calls}`, `BeforeTurnStartReplace{state_patch}`, `AfterTurnEndReplace{assistant_text, follow_up_messages}`, `OnErrorReplace{recovery, synthetic_response, backoff_ms}`. Validate at projection time; reject malformed with fail-safe-original semantics matching `_apply_prompt_transform`.
6. Composition policy: document and enforce per-event ordering for multiple replace hooks. `_apply_prompt_transform` today is last-write-wins by priority and name iteration; F-MUT extensions should adopt the same rule and surface it in the Author wizard so authors can reason about stacking order.
7. Permission boundary precedence: keep the existing rule that `final_action='permission_decision'` with `decision='deny'` wins over `replace`. The bus break order already enforces this today by virtue of block winning first; make it explicit in the new projector dispatch table so reviewers do not have to infer it from call ordering.

## Minimal extension shipped in this PR

Extension #5 (typed payload schemas) lands here as a pure additive seam: `magi_agent/hooks/replace_payloads.py` defines `BeforeToolUseReplace`, `AfterToolUseReplace`, `BeforeLlmCallReplace`, `AfterLlmCallReplace`, `BeforeTurnStartReplace`, `AfterTurnEndReplace`, `OnErrorReplace`, the `REPLACE_PAYLOAD_BY_POINT` table, and a `coerce_replace_payload(point, value)` helper that mirrors `_apply_prompt_transform`'s fail-safe-original semantics (malformed value → `None`, caller falls back to original payload). No emitter is wired in this PR — `tests/hooks/test_replace_payloads.py::TestNoConsumerWiredYet` pins that no other runtime module imports the helper, so runtime behavior is byte-identical. Wiring lands per-event in F-MUT1 (tool args) / F-MUT2 (LLM response and beyond) so the wizard's authoring kinds can already reference stable schema names without producing dead UI bindings.

## Sales narrative

The runtime difference that makes the F-MUT series possible: magi accepts `replace` at the hook emitter and projects it onto a real runtime payload, while Claude Code's hook contract is gate plus append only.
